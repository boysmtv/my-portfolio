# 6. MOTION LANGUAGE

## 6.1 Philosophy

Motion di website ini bukan "hiasan." Motion adalah **komunikasi**.

| Tipe | Fungsi | Contoh |
|------|--------|--------|
| **Orienting** | Memberi tahu user "kamu di sini" | Scroll progress, active nav |
| **Feedback** | "Aksi kamu diterima" | Button click, hover, focus |
| **Narrative** | "Cerita berlanjut" | Section reveal, scroll |
| **Ambient** | "Website ini hidup" | Background breathing, particles |
| **Transition** | "Konteks berubah" | Page transition, modal open |

## 6.2 Motion Tokens

```yaml
easing:
  production: [0.16, 1, 0.3, 1]    # Primary — custom cubic-bezier
    # Smooth, natural, tidak linear. Untuk entry, scroll reveal.
  expressive: [0.34, 1.56, 0.64, 1] # Spring-like bounce
    # Untuk micro-interactions, hover effects.
  exit: [0.55, 0, 1, 0.45]         # Fast exit
    # Untuk modal close, element remove.
  linear: [0, 0, 1, 1]              # Jangan dipakai.
    # Larangan total. Tidak natural.

duration:
  micro: 150ms     # Hover, focus, small state change
  fast: 300ms      # Button click, icon rotation
  normal: 500ms    # Section entry, card reveal
  slow: 800ms      # Hero entry, page transition
  ambient: 3000ms+ # Background animation cycle

spring:
  stiff: { stiffness: 300, damping: 30 }    # Buttons
  bouncy: { stiffness: 200, damping: 15 }   # Cards
  smooth: { stiffness: 150, damping: 20 }   # Modal
  gentle: { stiffness: 100, damping: 25 }   # Ambient
```

## 6.3 Animation Types

### Entry Animation
| Property | Value |
|----------|-------|
| Purpose | Elemen masuk ke viewport |
| Easing | `production` — [0.16, 1, 0.3, 1] |
| Duration | 500ms (normal) |
| Trigger | Scroll into viewport (once) |
| Variants | clip-up, fade-slide-up, scale-in, stagger-children |
| Frequency | Setiap section baru |
| Kapan stop | Setelah semua elemen terlihat |

### Hover Animation
| Property | Value |
|----------|-------|
| Purpose | Feedback bahwa elemen interactive |
| Easing | `expressive` — [0.34, 1.56, 0.64, 1] |
| Duration | 200-300ms |
| Trigger | Mouse enter |
| Variants | translateY(-2px), scale(1.02), shadow increase, icon slide |
| Frequency | Setiap interactive element |
| Kapan stop | Mobile/touch — gunakan tap state |

### Click Animation
| Property | Value |
|----------|-------|
| Purpose | Feedback aksi diterima |
| Easing | `spring(stiff)` |
| Duration | 150ms press, 300ms release |
| Trigger | Mouse down → up |
| Variants | scale(0.97) → bounce back, ripple |
| Frequency | Setiap clickable element |
| Kapan stop | Jangan berlebihan — hanya visible feedback |

### Layout Animation
| Property | Value |
|----------|-------|
| Purpose | Transisi layout yang smooth |
| Easing | `production` |
| Duration | 500ms |
| Trigger | State change (filter, sort, toggle) |
| Variants | `layout` prop, AnimatePresence, shared layout |
| Frequency | Saat user interaction |
| Kapan stop | Layout sudah stabil |

### Page Transition
| Property | Value |
|----------|-------|
| Purpose | Navigasi antar halaman terasa seamless |
| Easing | `production` |
| Duration | 400ms exit, 500ms enter |
| Trigger | Route change |
| Variants | Crossfade, slide, shared element |
| Frequency | Setiap navigasi |
| Kapan stop | Instant navigation (back/forward) |

## 6.4 Kapan Motion TIDAK Digunakan

1. **Preferred-reduced-motion** — User meminta reduced motion → semua animasi jadi fade transition 200ms
2. **Loading > 3 detik** — Skip entry animation, langsung tampilkan konten
3. **Scroll terlalu cepat** — Jika user scroll cepat, skip animation antara section
4. **Battery saving mode** — Kurangi atau nonaktifkan ambient animation
5. **Print mode** — Semua animation dinonaktifkan

# 7. DESIGN LANGUAGE

## 7.1 Spacing System

```yaml
spacing:
  unit: 4px  # Base unit — semua spacing kelipatan 4

  scale:
    xs: 4px    # 1 unit — micro spacing
    sm: 8px    # 2 unit — tight spacing
    md: 16px   # 4 unit — base spacing
    lg: 24px   # 6 unit — section padding
    xl: 32px   # 8 unit — large spacing
    '2xl': 48px # 12 unit — section gap
    '3xl': 64px # 16 unit — hero padding
    '4xl': 96px # 24 unit — page margins

  usage:
    button_padding: '{sm} {md}'         # 8px 16px
    card_padding: '{md} {lg}'           # 16px 24px
    section_padding_y: '{3xl}'          # 64px top/bottom
    section_gap: '{2xl}'                # 48px antar section
    content_max_width: '1200px'         # bukan max-w-7xl (1280px)
    paragraph_gap: '{md}'               # 16px antar paragraf
    heading_margin_bottom: '{sm}'       # 8px bawah heading
```

**Aturan**: Tidak boleh ada spacing yang bukan kelipatan 4px. Tidak boleh ada `space-y-5`, `space-y-7`, `space-y-10` — semuanya dari sistem.

## 7.2 Radius System

```yaml
radius:
  none: 0px        # Image, full-width elements
  sm: 4px          # Small UI elements
  md: 8px          # Default
  lg: 12px         # Cards
  xl: 16px         # Large cards
  '2xl': 20px      # Modals, sections
  full: 9999px     # Buttons, badges, pills

  usage:
    button: '{full}'
    card: '{lg}'           # 12px — lebih modern dari 16px
    modal: '{xl}'           # 16px
    section: '{2xl}'        # 20px
    input: '{md}'           # 8px
    badge: '{full}'
    image: '{lg}'
```

**Aturan**: Maksimal 3 radius berbeda dalam satu halaman. Saat ini ada 6+ radius berbeda — itu chaos.

## 7.3 Elevation & Shadow System

```yaml
elevation:
  0: 'none'
    # Text, flat elements
  
  1: '0 1px 2px rgba(0,0,0,0.3), 0 1px 3px rgba(0,0,0,0.15)'
    # Cards default, buttons rest
  
  2: '0 4px 6px rgba(0,0,0,0.3), 0 2px 4px rgba(0,0,0,0.2)'
    # Cards hover, dropdown

  4: '0 20px 25px rgba(0,0,0,0.4), 0 10px 10px rgba(0,0,0,0.2)'

glow:
  blue: '0 0 20px rgba(56,189,248,0.15), 0 0 40px rgba(56,189,248,0.08)'
    # For blue accent elements
    
  emerald: '0 0 20px rgba(16,185,129,0.15), 0 0 40px rgba(16,185,129,0.08)'
```

**Aturan**: Tidak boleh ada `shadow-[arbitrary_value]` di komponen. Semua shadow dari elevation system.

## Type Scale

```yaml
scale:
  xs: 0.75rem    # 12px — Captions, metadata
  sm: 0.875rem   # 14px — Body small, tags
  base: 1rem     # 16px — Body default
  lg: 1.125rem   # 18px — Body large, intro
  xl: 1.25rem    # 20px — Small heading
  '2xl': 1.5rem  # 24px — H3, section title
  '3xl': 2rem    # 32px — H2, section heading
  '4xl': 2.5rem  # 40px — H1, page heading
  '5xl': 3.25rem # 52px — Hero heading
  '6xl': 4rem    # 64px — Hero display (jika diperlukan)

hierarchy:
  h1:

    size: '{4xl}' | '{5xl}' (hero)

  h2:
    font: 'Cabinet Grotesk'
    size: '{3xl}'
    weight: 700
    tracking: '-0.01em'
    leading: '1.15'
    
  h3:
    font: 'Cabinet Grotesk'
    size: '{xl}'
    weight: 600
    tracking: '0'
    leading: '1.2'
    
  body:
    font: 'Inter'
    size: '{base}' | '{lg}' (featured)
    weight: 400
    tracking: '0'
    leading: '1.7'
    
  caption:

    size: '{xs}'
    weight: 500
```

## Usage Rules

1. **Satu h1 per halaman** — di Hero. Tidak boleh lebih.
2. **Heading tidak boleh wrap ke 3+ baris** — Jika wrap, kecilkan font atau pendekkan text.
3. **Body text tidak boleh lebih dari 75 karakter per baris** — Ideal 60-70 karakter.
4. **Jangan gunakan font-weight 300 untuk body** di dark mode — terlalu tipis, susah dibaca.
5. **Letter-spacing negatif** untuk heading besar (+40px) — membuat lebih tight dan premium.
6. **Tidak boleh ada underline** di heading — hanya link body yang boleh underline.

# 15. WOW MOMENTS

## 20+ Momen yang Membuat Pengunjung Terkesan

### Loading & Entry
1. **Noise texture** — Tidak kelihatan secara sadar, tapi membuat background terasa "kaya" dan "mahal."
2. **Entry sequence** — Element muncul staggered dengan timing presisi — terasa choreographed, bukan random.
3. **Photo reveal** — Foto muncul dengan subtle scale-in dan warm glow — langsung establishing personal connection.

### Hero
4. **Headline** — "Building payment systems that survive production reality" — the word "survive" is unexpected.
5. **Trust bar count-up** — Angka bergerak dari 0 ke nilai final — membuat data terasa hidup.
6. **Flow diagram background** — Subtle lines bergerak sangat lambat — "Ini punya motif, bukan random."

### Navigation
7. **Scroll progress** — Garis tipis di navbar yang mengikuti scroll — small detail that signals quality.
8. **Active section highlight** — Nav link berubah saat scroll — "Website ini tahu saya di mana."

### Project Showcase
9. **Flagship project scale** — Project pertama tampil lebih besar, lebih detail dari yang lain — hierarchy yang jelas.
10. **Evidence panels** — "Operational pressure", "Constraint handled", "Why it mattered" — framework yang tidak biasa.
11. **Accent gradient** — Subtle gradient overlay di project card — warna berbeda per project.
12. **ArrowUpRight animation** — Icon bergerak saat hover — micro-interaction yang bikin card terasa alive.

### Case Study
13. **Problem/Approach/Tradeoff/Impact grid** — Four-column layout yang langsung menunjukkan structured thinking.
14. **Tradeoff section** — Most portfolios only show wins. Showing tradeoffs = maturity signal.

### Experience
15. **Timeline connector line animation** — Garis menghubungkan dots dari atas ke bawah saat scroll — satisfying.
16. **Signal tags** — Specific, relevant keywords per role — bukan generic responsibilities.

### Architecture
17. **Interactive diagram** — Node yang bisa di-click, connector lines — "Ini bukan sekedar text cards."

### Capabilities
18. **Custom icons** — Bukan Lucide generik — icons yang spesifik untuk technology stack.

### Final CTA
19. **Background shift to emerald** — Warna berubah dramatic — "This is the moment."
20. **"If the problem is delivery under pressure, let's talk"** — Specific CTA. Bukan "Contact me."

### Micro-interactions (Distributed)
21. **Button spring bounce** — Click terasa memuaskan karena spring physics.
22. **Card lift** — Hover: card naik 4px dengan shadow meningkat — tactile feel.
23. **Modal smooth open** — Scale + fade + backdrop — terasa premium, tidak popup biasa.
24. **Tag hover scale** — Chips membesar sedikit saat hover — interactive feel.
25. **Page transition** — Smooth crossfade saat navigasi ke case study page.

# 16. ANTI-PATTERNS

## Daftar Larangan Mutlak

### Visual
1. **❌ Stock photos or generic illustrations** — Setiap visual harus original atau meaningful.
2. **❌ Gradient text in headings** — Tren 2023 yang sudah mati. Terlihat murahan.
3. **❌ Animated gradient backgrounds** — Kecuali sangat subtle (< 5s cycle).
4. **❌ Particle effects** — Terlalu 2020. Kecuali dalam konteks yang sangat spesifik.
5. **❌ Heavy glassmorphism** — Glass untuk navbar dan modal saja. Jangan untuk cards.
6. **❌ Multiple glow overlays** — Satu glow di Hero cukup. Jangan tumpuk.
7. **❌ Box shadows yang berbeda-beda** — Semua shadow dari elevation system.

### Motion
8. **❌ Animasi tanpa tujuan** — Setiap animasi harus menjawab: "Apa yang ini komunikasikan?"
9. **❌ Entrance animation yang terlalu lama** — > 800ms untuk entry = annoying.
10. **❌ Same animation for everything** — Variasi entry, hover, scroll, exit.
11. **❌ Autoplay video / audio** — Never. User harus initiate.
12. **❌ Horizontal scroll** — Kecuali interactive timeline yang explicit.
13. **❌ Paralaks di semua section** — Hanya untuk Hero dan mungkin Case Study.

### Content
14. **❌ "I'm passionate about..."** — Tunjukkan, jangan katakan.
15. **❌ Skill progress bars** — "Kotlin 90%" is meaningless and cringe.
16. **❌ Klise portfolio copy** — "I create amazing user experiences" — delete.
17. **❌ Too many technologies listed** — Focus on depth, not breadth.
18. **❌ Fake metrics** — Jangan bikin angka. Pakai data real atau jangan.

### UX
19. **❌ Link ke LinkedIn sebagai CTA** — Jangan arahkan user keluar website untuk hal yang bisa dilakukan di sini.
20. **❌ Contact form yang panjang** — Nama, email, message sudah cukup. Jangan minta phone, company, dll.
21. **❌ Social media icons di hero** — Simpan di footer. Hero adalah tentang user, bukan tentang social proof.
22. **❌ Auto-play carousel** — User harus kontrol.
23. **❌ "Skip" link yang tidak visible** — Skip to content harus visible on first tab.

### Layout
24. **❌ Single column untuk semuanya** — Variasi layout antar section.
25. **❌ Too many sections** — Maksimal 9 section (saat ini 16 — terlalu banyak).
26. **❌ Infinite scroll** — There must be an end.
27. **❌ Fixed background attachment** — Performa berat. Gunakan scroll position.

### Technical (UX Level)
28. **❌ Links that open in same tab for external sites** — External = new tab.
29. **❌ Mailto sebagai primary contact method** — Mailto fine, tapi juga sediakan form atau copy-able email.
30. **❌ No focus indicators** — Every interactive element must have focus state.

# 18. FINAL EXPERIENCE MANIFESTO

## The Production-Grade Portfolio Manifesto

**Setiap portfolio engineer bisa bilang "saya build ini."**
**Portfolio ini ada karena "saya operasikan itu."**

---

1. **Konten adalah raja, tetapi visual adalah kerajaan.**
   Konten kuat tanpa visual adalah buku di ruang gelap. Visual kuat tanpa konten adalah poster kosong. Keduanya harus setara.

2. **Satu bukti lebih kuat dari seratus klaim.**
   Satu "99.9% uptime maintained" lebih baik dari "I'm a reliable engineer." Tunjukkan angkanya, jangan katakan sifatnya.

3. **Setiap pixel punya alasan.**
   Tidak ada random spacing. Tidak ada random color. Tidak ada random animation. Jika tidak bisa menjelaskan kenapa sesuatu ada, hapus.

4. **Tenang itu lebih kuat dari heboh.**
   Website ini tidak perlu berteriak. Tidak perlu animasi bombastis. Tidak perlu efek yang menarik perhatian secara paksa. Ketenangan adalah tanda kepercayaan diri.

5. **Konsistensi adalah premium.**
   Mahal bukan karena bahan bakunya, tapi karena ketelitiannya. Radius yang konsisten, spacing yang konsisten, warna yang konsisten — itulah yang membuat website terasa "Apple-like."

6. **Gerakan harus berarti.**
   Setiap animasi berkomunikasi: "kamu di sini," "aksi diterima," "cerita berlanjut." Jika sebuah animasi tidak mengkomunikasikan apa pun, ia hanya noise.

7. **Portfolio ini adalah produk.**
   Bukan resume online. Bukan CV interaktif. Produk yang dirancang untuk menyelesaikan masalah: "kenapa saya harus hire orang ini?"

8. **Evidence over opinion.**
   Bukan "saya percaya..." tapi "saya telah..." Bukti lebih meyakinkan daripada keyakinan.

9. **Kualitas konten harus setara dengan kualitas visual.**
   Copywriting yang kuat membutuhkan visual yang kuat. Visual yang kuat membutuhkan copywriting yang kuat. Satu tidak bisa menyelamatkan yang lain.

10. **Redesign ini bukan soal "lebih cantik."**
    Redesign ini soal "lebih jelas, lebih meyakinkan, lebih berkesan." Cantik adalah efek samping, bukan tujuan.
